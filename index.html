O que este projeto faz

Você informa uma URL e recebe um resumo estruturado com recomendações iniciais. Neste Sprint, a análise é mockada para validar interface, estado e acessibilidade antes da integração real (CORS / proxy no Sprint 2).

  • Validação de URL com new URL()
  • Estados: idle / loading / error / success
  • Acessibilidade: foco, aria-live, teclado
  • Layout responsivo e componentes reutilizáveis
